# Test Review Skill

## Purpose

Review whether the implementation has enough validation evidence.

## Review Rules

Check whether:

1. Relevant tests were run.
2. Test output is included as evidence.
3. Failing tests are not ignored.
4. New behavior has matching tests.
5. Regression risk is considered.
6. Manual validation is clearly separated from automated validation.

## Output Requirements

The review must state:

- tests reviewed
- evidence path or command output
- missing validation
- risk level
- pass/fail/block decision
